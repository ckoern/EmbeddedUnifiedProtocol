# Embedded Unified Protocol (EUP) — framing, commands & streams

A small, allocation-free C++17 protocol stack for an embedded USB device that
exchanges **commands**, **replies**, and unsolicited **stream** packets with a
host. On top of the framing layer (COBS + CRC16) it adds typed **command
dispatch**, a symmetric **host call** API, and one-way **stream** packets — all
allocation-free and driven by the same compile-time codecs. Use just the framing
layer, or the whole stack.

## Showcase

**Shared contract** — one header both ends include:

```cpp
// commands.hpp
#include "eup/command.hpp"
using namespace eup;

std::tuple<StatusCode, std::uint16_t> add(std::uint8_t a, std::uint8_t b);
std::tuple<StatusCode>                set_led(bool on);

using AddCmd    = CommandDef<0x01, decltype(add)>;
using SetLedCmd = CommandDef<0x02, decltype(set_led)>;
```

**Device** — implement the handlers, register them, dispatch received frames:

```cpp
#include "commands.hpp"

std::tuple<StatusCode, std::uint16_t> add(std::uint8_t a, std::uint8_t b) {
    return {StatusCode::Ok, static_cast<std::uint16_t>(a + b)};
}
std::tuple<StatusCode> set_led(bool on) {
    gpio_write(LED, on);
    return {StatusCode::Ok};
}

constexpr std::array<CommandEntry, 2> kCommands{{
    command<AddCmd,    &add>(),
    command<SetLedCmd, &set_led>(),
}};

// for each decoded Command frame `cmd`:
std::uint8_t reply[kMaxPayload];
std::uint8_t replyLen = 0;
dispatch(kCommands, cmd, reply, sizeof(reply), replyLen);
// then frame + send: encode_frame(MessageType::Reply, reply, replyLen, ...)
```

**Host** — call a command and get the typed result back (transport omitted):

```cpp
#include "commands.hpp"
#include "eup/host.hpp"

auto [status, sum] = call<AddCmd>(transport, std::uint8_t{20}, std::uint8_t{22});
// status == StatusCode::Ok, sum == 42
```

## Wire format

A packet is at most **256 bytes**:

```
+----------+------+--------+-------------------+-----------+--------+
| overhead | TYPE | LENGTH | PAYLOAD (0..250)  | CRC16 (2) | 0x00   |
|  (COBS)  |      |        |                   |  LE       | delim  |
+----------+------+--------+-------------------+-----------+--------+
\__________________ COBS-encoded region ___________________/
```

- **0x00 delimiter** — the only zero byte on the wire; it **trails** each
  packet, marking the frame boundary. The receiver splits the stream on `0x00`,
  so the delimiter that closes one frame also separates it from the next.
- **COBS region** — Consistent Overhead Byte Stuffing of the raw block
  `TYPE | LENGTH | PAYLOAD | CRC16`. For our sizes (raw ≤ 254 bytes) COBS adds
  exactly one overhead byte.
- **TYPE** — message type (`Command`, `Reply`, `Stream`, …); opaque to the codec.
- **LENGTH** — payload length, `0..250`.
- **PAYLOAD** — `LENGTH` bytes; may contain `0x00` (COBS handles it).
- **CRC16** — CRC-16/CCITT-FALSE over `TYPE + LENGTH + PAYLOAD`, **little-endian**.

### Size budget (worst case)

| Part        | Bytes |
|-------------|-------|
| TYPE+LENGTH | 2     |
| PAYLOAD     | 250   |
| CRC16       | 2     |
| raw block   | 254   |
| + COBS overhead | 1 |
| + delimiter | 1     |
| **on wire** | **256** |

## Design choices

- **No dynamic allocation, no exceptions, no RTTI** — caller-provided buffers,
  status-code returns, fixed-size structs. Builds with `-fno-exceptions
  -fno-rtti`.
- **CRC-16/CCITT-FALSE**: poly `0x1021`, init `0xFFFF`, no reflection, xorout
  `0x0000` (check value `0x29B1`). Transmitted little-endian. To switch
  endianness, change the two byte writes in `encode_frame` and the read in
  `decode_region`.
- **Streaming receive**: `FrameReader` splits the incoming byte stream on `0x00`
  and decodes each frame, so unsolicited `Stream` packets interleaved with
  `Reply` packets are handled naturally.

## Usage

The stack is built in layers; include only what you need.

| Header | Provides |
|--------|----------|
| `eup/frame.hpp`   | Framing: `encode_frame`, `decode_region`, `FrameReader` |
| `eup/codec.hpp`   | Typed (de)serialization: `Codec<T>`, `InlineArray`/`InlineString` |
| `eup/status.hpp`  | `StatusCode` (framework + application codes) |
| `eup/command.hpp` | Command contracts + device-side dispatch |
| `eup/host.hpp`    | Symmetric host call API |
| `eup/stream.hpp`  | Unsolicited device → host stream packets |

The snippets below assume `using namespace eup;`. A complete runnable host +
device program is in [`examples/rgb_led_adc`](examples/rgb_led_adc).

### Framing

```cpp
#include "eup/frame.hpp"

// --- encode ---
std::uint8_t wire[kMaxPacket];
const std::uint8_t params[] = {0x10, 0x20};
auto [status, len] = encode_frame(MessageType::Command, params, 2,
                                  wire, sizeof(wire));
// status == FrameStatus::Ok; transmit wire[0 .. len)

// --- decode (streaming) ---
FrameReader reader;
for (std::uint8_t b : received_bytes) {
    switch (reader.push(b)) {
        case FrameReader::Event::FrameReady: {
            const Frame& f = reader.frame();
            // dispatch on f.type, f.payload[0 .. f.length)
            break;
        }
        case FrameReader::Event::Error:
            // reader.lastStatus() == CrcMismatch / BufferTooSmall / ...
            break;
        case FrameReader::Event::NeedMore:
            break;
    }
}
```

`decode_region` is also available when you have already split the stream on
`0x00` yourself.

### Commands (request / reply)

A command can be implemented via a function with the signature
`std::tuple<StatusCode, Results...>(Arguments...)` — a leading `StatusCode`
followed by any result values, taking any argument values. The opcode and wire
format live in a **contract** shared by both ends.

```cpp
// shared header — handler declaration + contract
std::tuple<StatusCode, std::uint16_t> read_adc(std::uint8_t channel);
using ReadAdcCmd = CommandDef<0x11, decltype(read_adc)>;
```

Device — implement the handler, register it, and dispatch received frames:

```cpp
std::tuple<StatusCode, std::uint16_t> read_adc(std::uint8_t channel) {
    return {StatusCode::Ok, sample_adc(channel)};
}

constexpr std::array<CommandEntry, 1> kCommands{{
    command<ReadAdcCmd, &read_adc>(),
}};

// `cmd` is a decoded Command frame (e.g. from FrameReader):
std::uint8_t reply[kMaxPayload];
std::uint8_t replyLen = 0;
dispatch(kCommands, cmd, reply, sizeof(reply), replyLen);
// frame the reply: encode_frame(MessageType::Reply, reply, replyLen, wire, sizeof(wire));
```

Host — call it like a local function over a transport:

```cpp
auto [status, value] = call<ReadAdcCmd>(transport, std::uint8_t{2});
// returns std::tuple<StatusCode, std::uint16_t> — the mirror of the handler
```

The transport is any type providing:

```cpp
bool send_command(const std::uint8_t* wire, std::size_t len) noexcept;
bool await_reply(Frame& out) noexcept;  // returns only the matching Reply frame
```

### Streams (unsolicited device → host)

A stream packet is one-way; its payload is `[opcode | uint32 counter | fields…]`.
The contract is types only:

```cpp
// shared header
using TelemetryStream = StreamDef<0x20, std::uint16_t, std::int16_t>;
```

Device — produce a packet (the `counter` value is application-supplied):

```cpp
std::uint8_t wire[kMaxPacket];
StreamWriter writer(wire, sizeof(wire));
auto [status, len] = writer.write<TelemetryStream>(seq, adc0, temp_c10);
// transmit wire[0 .. len)
```

Host — register a handler and route received `Stream` frames to it:

```cpp
void on_telemetry(std::uint32_t counter, std::uint16_t adc0, std::int16_t temp);

constexpr std::array<StreamEntry, 1> kStreams{{
    stream<TelemetryStream, &on_telemetry>(),
}};

dispatch_stream(kStreams, streamFrame);  // decodes counter + fields, calls the handler
```

### Typed payloads & spans

Command arguments/results and stream fields are serialized by `Codec<T>`, which
covers the fixed-width integers, `bool`, `float`/`double`, and enums
(little-endian). For short, variable-length data use the owning spans:

```cpp
InlineString<16>              name;     // length-prefixed byte string
InlineArray<std::uint16_t, 8> samples;  // up to 8 little-endian uint16
```

A span works anywhere a field does — command argument, result, or stream field.
Add a new transmittable type by specializing `Codec<T>`.

Worst-case payload sizes are checked at **compile time**: a command or stream
whose arguments/results/fields cannot fit in a 256-byte packet fails to build.

## Building the host tests

```
cmake -S . -B build
cmake --build build
ctest --test-dir build --output-on-failure
```

The tests use [Catch2 v3](https://github.com/catchorg/Catch2), pulled
automatically at configure time via CMake `FetchContent` (or reused if already
installed) — nothing to install by hand. Each `TEST_CASE` is registered as its
own `ctest` test.

The library sources (`src/*.cpp` + `include/`) compile directly into firmware
with any C++17 toolchain — no CMake, Catch2, or host dependencies required.
